Skill system

Skills are files, not code

How a skill loads

1
always-on — cr-evaluator loads on every request. Rubric, module map, complexity guide injected into system prompt unconditionally.
2
semantic detection — Haiku reads skill descriptions and user intent. "write a technical architecture doc" → loads SDD skill even without the word "sdd".
3
keyword fallback — if Haiku fails, trigger keywords catch common cases. Graceful degradation.
4
activate_skill tool — Sonnet can call this during the loop. Last resort, agent-driven. Skill content returned as a tool result.

Research tools

Searches before every answer — no training-data guesses

Jira

get_jira_ticket · search_jira

Fetches specific tickets by ID or searches by keyword. Converts Atlassian Document Format (ADF) to clean plain text. Used to find precedent CRs and related implementations.

Confluence

search · get_page

Full-text search across solution docs and architecture pages. Fetches complete page content when a result looks relevant. Includes circuit-breaker retry logic.

Kapa Docs

search_kapa_docs

Searches Capillary's internal documentation via the Kapa MCP endpoint. Optional — degrades gracefully if CAPILLARY_DOCS_MCP_URL is not set.

Docs Sitemap

search_docs_site

No search API needed. Fetches docs.capillarytech.com/sitemap.xml once, scores URLs by keyword overlap, fetches top 3 pages. Cached with 1hr TTL.

All tools are conditional on env vars at startup · handlers never throw · errors return {"error":"...", "partial":true}

Engineering optimisations

Four fixes that changed the experience

🔇

No narration snowball

Each multi-turn response used to repeat all previous narration — "I will do X. I did X. Now I will do Y..." Fixed by stripping text blocks from tool-calling turns in cleanBlocks before pushing to messages. Model never sees its own narration between turns.

⚡

Parallel Haiku summarisation

Tool results were summarised sequentially — N tools × ~1s per Haiku call. Changed to Promise.all so all summaries fire simultaneously. Also passes problemText context so Haiku preserves the relevant details, not random ones.

🧩

Semantic skill detection

Replaced keyword trigger matching with a Haiku sub-agent that reads skill descriptions and understands intent. "Write a technical architecture document" now loads the SDD skill even without the exact word "sdd". Keyword match retained as fallback.

📊

Understand phase parallism

Classification and skill detection used to run sequentially before the agent loop. Both are now launched with Promise.all — two Haiku calls happening at the same time. Removes 300–500ms from every request's understand phase.
