Layer 1: Gatherer Agents

Always-on, real-time signal collection from every client touchpoint

Gatherer agents operate 24/7, monitoring all client communication channels in real-time. When a signal arrives — a client email, a Jira comment, a Confluence update, a meeting transcript — the agent indexes it immediately and nudges the Processor Agent Manager with a structured change event.

📧
Email Agent — Forward any client email to the platform. The agent extracts requirements, sentiment, and action items. No login needed — just forward and it's processed.
🎫
Jira Watcher — Real-time webhooks on ticket creation, status changes, and comments. Detects requirement changes, priority shifts, and blockers automatically.
📝
Confluence Listener — Monitors page edits and new documents. When a solution doc changes, downstream processors are notified within seconds.
🎙️
Meeting Transcriber — Processes meeting recordings and extracts decisions, action items, and sentiment shifts. Links findings to relevant projects.
💬
Slack Monitor — Watches project channels for client mentions, escalation signals, and informal requirement changes that never make it to Jira.
🔗
CRM Sync — Pulls deal stage changes, renewal dates, and relationship health scores. Feeds into resource planning and sentiment analysis.

Layer 2: Processor Agents + Impact Dependency Graph

Intelligent change propagation — one signal updates all affected documents

The Processor Agent Manager maintains an Impact Dependency Graph — a directed graph that maps which data sources affect which derived documents. When a change signal arrives from the Gatherer layer, the Manager traverses the graph to determine exactly which processing agents need to activate.

Processing Agents

📋 BRD Agent

Maintains living Business Requirements Documents. Auto-updates when new requirements are detected from any source.

📐 SDD Agent

Keeps Solution Design Documents current. Propagates requirement changes into technical architecture decisions.

💚 Sentiment Agent

Continuously scores client sentiment from emails, meeting tones, and Jira comment patterns. Alerts on negative trends.

📊 Gap Analyzer

Compares current BRD against Capillary capabilities. Scores each domain and identifies implementation gaps in real-time.

🔢 Integration Scorer

Calculates integration complexity and health scores. Tracks API usage, error rates, and configuration drift.

👥 Resource Allocator

Matches projects to people based on experience, bandwidth, and domain expertise. Recommends optimal assignments.

📊

Resource Intelligence & Utilization

Smart allocation based on experience, bandwidth, and client needs

Resource Intelligence is deeply integrated into both the Processor and Consumer layers. It continuously tracks team capacity from meetings, Jira activity, and Confluence contributions — then makes intelligent assignment recommendations.

🎯
Experience-based matching — A PM with 3 years of retail loyalty experience gets recommended for new retail projects. Domain expertise is tracked and weighted.
📈
Bandwidth analysis — Real-time utilization calculated from meeting load, active Jira tickets, and Confluence activity. Prevents overallocation before it happens.
🔄
Sentiment-driven reassignment — When client sentiment drops below threshold, the system can recommend PM or SA reassignment with full context handoff. Human approval required.
📋
Utilization dashboards — Leadership gets real-time visibility into team capacity, project health, and resource bottlenecks — without asking anyone for a status update.
🧑‍💼

Layer 3: Consumer Agents

Persona-aware AI assistants that serve the right intelligence to the right person

Consumer Agents are the user-facing layer. Each agent understands the persona of the person asking — a CSM gets sentiment and relationship insights, a Sales rep gets gap analysis and upsell opportunities, an SA gets technical feasibility and architecture guidance.

💚 CSM Agent

Serves: Client sentiment trends, relationship health, risk alerts, renewal readiness. Nudges: "Client X sentiment dropped 15% this week — 3 unresolved tickets flagged."

💰 Sales Agent

Serves: Implementation gaps, upsell opportunities, competitive positioning. Nudges: "Client Y using only 40% of licensed modules — loyalty+ and analytics are natural upsells."

🔧 SA Agent

Serves: Technical feasibility, architecture recommendations, effort estimates. Nudges: "Similar integration done for Client Z in Q2 — reusable components available."

📋 PM Agent

Serves: Project health, timeline risks, resource conflicts, dependency tracking. Nudges: "Sprint velocity dropped 30% — 2 blockers need escalation."

👨‍💻 Engineering Agent

Serves: Code context, API docs, test coverage gaps, deployment readiness. Nudges: "Integration test coverage for Module X at 45% — 3 critical paths untested."

👔 Leadership Agent

Serves: Portfolio health, resource utilization, revenue risk, strategic insights. Nudges: "3 accounts showing declining sentiment — combined ARR at risk: $2.1M."

Guardrails & Human-in-the-Loop

Every critical decision has a human checkpoint. AI does the work — humans make the decisions.

🛡️

Approval Gates

No document is published, no code deployed, no resource reassigned without explicit human approval via email or Slack.

🔒

Confidence Thresholds

Low-confidence outputs flagged for human review. High-confidence routine updates proceed autonomously — humans focus on exceptions.

👁️

Full Transparency

Every agent action logged with reasoning. Humans can inspect why a decision was made and override at any point.

⚡

Kill Switch

Any pipeline can be paused or terminated instantly. Agents respect scope boundaries and cannot access unauthorized systems.

📏

Quality Validation

Every generated artifact passes validation rules before reaching a human. Malformed outputs are caught and regenerated automatically.

🔐

Role-Based Access

Agents inherit persona permissions. A Sales agent cannot access engineering systems. A PM agent cannot approve deployments.
