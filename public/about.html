Capillary Solution Agent

AI-powered feasibility assessment for the Customer Success team

● Production Anthropic Claude Multi-user Auth LangGraph

What it does

  • CS engineers submit Jira tickets, describe client requirements, or upload documents
  • Agent searches Jira, Confluence, Kapa docs, and docs.capillarytech.com for precedents and documentation
  • Returns a structured feasibility assessment: Problem → Verdict → Approach → Complexity → References → Open Questions
  • Flags for SA escalation when the request exceeds routine evaluation scope

Architecture

Web UI built on Express with SSE streaming. The backend runs a LangGraph state machine that routes requests through a multi-step pipeline.

Request pipeline

classify → loadSkills → skillRouter
single-mode: research → validate
multi-node: researchFanOut → skillValidate

Single-mode path

Standard feasibility assessments. One Sonnet streaming call handles research and synthesis in a multi-turn agentic loop.

Multi-node path

SDDs and gap analyses. Parallel research across tool categories, then parallel section writing — each section by the right model.

Auth

Multi-user with bcrypt + express-session. Google and Microsoft SSO supported. Session-based, cookie-secured.

Client persona

Auto-detected from problem text and injected into the system prompt so responses are contextualised to the client's stack.

Skills system

Skills are prompt bundles in skills/ folders, not code. Adding a skill means creating a folder and one line in registry.json — zero JS changes.

🎯 cr-evaluator

Always-on feasibility rubric. Loaded on every request — defines OOTB/Config/Custom/Not Feasible verdicts, module map, and complexity scoring.

📄 capillary-sdd-writer

Generates Solution Design Documents. Multi-node execution with parallel research and section writing. Output delivered as a downloadable file.

📊 solution-gap-analyzer

BRD gap analysis with scoring. Multi-node execution. Compares client requirements against platform capabilities and scores each gap.

🖼️ excalidraw-diagram

Visual architecture diagrams. Generates Excalidraw JSON for solution architecture visualisation.

Skills have manifests (manifest.json) declaring execution mode, research phases, synthesis sections, and validation rules. Document-class skills deliver output as downloadable files, not raw chat text.

Research tools

Source Tools Notes
Jira get_jira_ticket, search_jira, add_jira_comment Fetches tickets, searches by keyword, posts comments. ADF → plain text conversion.
Confluence search_confluence, get_confluence_page, create_confluence_page Full-text search, page retrieval, page creation. Circuit-breaker retry logic.
Kapa Docs search_kapa_docs MCP protocol — semantic search over Capillary documentation. Degrades gracefully if unconfigured.
Web Search search_docs_site Sitemap-based search of docs.capillarytech.com. No external API — scores URLs by keyword overlap, fetches top pages.

Model strategy

Every operation uses the cheapest model that can do the job correctly. Sonnet tokens are spent only on tasks that require multi-source reasoning.

Sonnet 4 Synthesis Main agentic loop (streaming), reasoning sections in document skills
Haiku 4.5 Classification Request classification, skill selection, tool result summarisation, structural sections, escalation summary

runSubAgent enforces model validation — only Sonnet 4 and Haiku 4.5 are allowed. MAX_AGENT_TOKENS applies only to the Sonnet streaming call.

Deployment
